# 水面光伏锚固系统数据标注作业指导书 (SOP)

**版本**: v1.0  
**生效日期**: 2026-02-27  
**适用范围**: 水面光伏区锚固系统图像标注项目  
**编制**: CV数据工程团队  

---

## 1. 项目概述

### 1.1 项目背景

水面光伏区锚固系统是维持浮体阵列稳定的关键基础设施。由于长期暴露在恶劣的水面环境中（紫外线、波浪冲击、藻类腐蚀等），锚固组件容易出现松动、磨损、断裂、腐蚀等缺陷。本标注规范旨在建立标准化的图像标注流程，为后续的AI检测模型训练提供高质量的数据基础。

### 1.2 场景特点

| 场景特征 | 描述 | 标注挑战 |
|---------|------|---------|
| **强反光** | 水面阳光反射导致局部过曝 | 目标轮廓模糊 |
| **波纹干扰** | 水波造成图像畸变和动态模糊 | 边缘定位困难 |
| **藻类覆盖** | 绿色藻类附着在组件表面 | 颜色特征改变 |
| **小目标** | 锚件细节在远距离拍摄下尺寸小 | 需保证≥50px |
| **遮挡** | 组件间相互遮挡或植被遮挡 | 部分不可见 |

### 2.2 部件定义与视觉特征

#### 2.2.1 锚绳 (Anchor Rope)

**物理描述**: 用于连接浮体与锚块的柔性缆绳，常见材质为钢丝绳或高分子纤维绳。

| 状态 | 视觉特征描述 | 判别标准 | 典型图像示例描述 |
|------|------------|---------|----------------|
| **normal** | 绳索紧绷，呈直线或规则曲线 | 下垂弧度 < 5cm（相对跨度） | 绳索平直连接浮筒与锚块，无松弛下垂 |
| **loose** | 绳索下垂明显，或表面磨损 | 下垂弧度 > 5cm；或可见磨损起毛 | 绳索中部明显下垂呈"V"字形，或表面纤维松散 |
| **missing** | 绳索完全不可见或断裂 | 连接点间无可见绳索；或明显断裂端点 | 仅余两端连接点，中间无连接；或断裂绳索漂浮水面 |
| **damaged** | 绳索局部破损、断股 | 可见内部钢丝断裂；或护套破损面积 > 10% | 绳索局部膨胀变形，可见内部结构外露 |

#### 2.2.2 连接件 (Anchor Connector)

**物理描述**: 连接锚绳与浮体/锚块的金属构件，包括卸扣、眼板、螺栓组等。

| 状态 | 视觉特征描述 | 判别标准 | 典型图像示例描述 |
|------|------------|---------|----------------|
| **normal** | 金属表面完整，连接紧密 | 无可见间隙；无锈蚀 | 金属表面光泽均匀，螺栓紧固无松动 |
| **loose** | 螺栓松动，或连接间隙可见 | 螺母与基座间隙 > 2mm | 螺母位置偏离，可见螺纹外露 |
| **missing** | 连接件完全脱落 | 安装位置空无一物 | 仅余安装孔洞，连接件缺失 |
| **damaged** | 金属腐蚀、变形或裂纹 | 锈蚀面积 > 10%；或明显变形 | 表面红褐色锈斑，或金属弯曲变形 |

#### 2.2.3 锚块 (Anchor Block)

**物理描述**: 沉于水底的混凝土或铸铁块，用于提供锚固力。

| 状态 | 视觉特征描述 | 判别标准 | 典型图像示例描述 |
|------|------------|---------|----------------|
| **normal** | 位置正确，外观完整 | 与设计位置偏差 < 10cm | 锚块位于指定水域，外观方正 |
| **loose** | 锚块移位，或倾斜 | 与设计位置偏差 > 10cm；或明显倾斜 | 锚块偏离原位置，绳索角度异常 |
| **missing** | 锚块沉没不可见或丢失 | 预期位置无锚块；或仅余连接绳索 | 水面无锚块迹象，绳索垂入水中 |
| **damaged** | 混凝土崩角、开裂 | 崩角尺寸 > 5cm；或裂缝宽度 > 2mm | 锚块边角缺损，或表面可见裂纹 |

#### 2.2.4 浮筒连接件 (Float Fastener)

**物理描述**: 连接相邻浮筒的构件，包括螺栓、卡扣、铰链等。

| 状态 | 视觉特征描述 | 判别标准 | 典型图像示例描述 |
|------|------------|---------|----------------|
| **normal** | 连接紧密，无位移 | 浮筒间缝隙均匀 < 3cm | 连接螺栓紧固，浮筒排列整齐 |
| **loose** | 连接松动，浮筒错位 | 浮筒间缝隙 > 3cm；或高低差 > 5cm | 浮筒间可见明显间隙，或高低不平 |
| **missing** | 连接件缺失 | 连接位置无螺栓/卡扣 | 连接孔洞空置，浮筒间无连接 |
| **damaged** | 连接件断裂、变形 | 可见裂纹或明显弯曲 | 金属连接件断裂，塑料件开裂 |

## 3. 边界情况判定规则 (Edge Cases)

### 3.1 锚绳特殊处理规则

#### 3.1.1 部分浸没处理

**场景**: 锚绳部分被水淹没，仅部分可见。

| 可见比例 | 处理规则 | 标注方式 | 示例 |
|---------|---------|---------|------|
| 可见 ≥ 80% | 正常标注 | 标注完整边界框 | 仅末端少量入水 |
| 50% ≤ 可见 < 80% | 标注可见部分 | 框选可见段，`difficult=True` | 中段部分浸没 |
| 可见 < 50% | 判断为断裂 | 标注为 `anchor_rope_missing` | 仅两端可见 |
| 完全不可见 | 标记缺失 | 标注 `anchor_rope_missing` | 完全沉入水中 |

**操作指引**:
1. 可见部分用矩形框紧密包围
2. 若中断超过 20% 绳长，优先标记为 `missing`
3. 在标签文件中添加 `difficult` 标记

#### 3.1.2 线段转矩形规范

原始标注可能以线段形式给出（两端点），需转换为YOLO格式矩形：

```
线段端点: (x1, y1) -> (x2, y2)
转换公式:
  x_min = min(x1, x2) - padding
  y_min = min(y1, y2) - padding
  x_max = max(x1, x2) + padding
  y_max = max(y1, y2) + padding
  
其中 padding = max(5px, 绳径/2)
```

**特殊情况**:
- 垂直绳：宽度设为 10-20px
- 水平绳：高度设为 10-20px
- 斜向绳：按实际角度计算外接矩形

### 3.2 反光干扰处理

**场景**: 水面强反光导致目标轮廓不清。

#### 3.2.1 决策流程

```
反光区域是否覆盖目标?
├── 是 → 能否通过上下文推断?
│       ├── 能 → 标注为 normal/对应状态，confidence降低
│       └── 否 → 标注为 difficult，添加反光标签
└── 否 → 正常标注
```

#### 3.2.2 具体规则

| 反光程度 | 可见度 | 处理策略 | 标记要求 |
|---------|--------|---------|---------|
| 轻微 | 轮廓可辨 | 正常标注 | 无特殊标记 |
| 中度 | 轮廓模糊 | 标注可见部分，`difficult=True` | 添加 `glare` 标签 |
| 重度 | 完全不可见 | 根据上下文推断或跳过 | 添加 `glare` + `occlusion` 标签 |

**标注建议**:
- 当反光导致边界不清时，适当扩大边界框（+10%）以包含不确定区域
- 在标注工具中添加备注说明反光情况

### 3.3 重叠遮挡处理

#### 3.3.1 多锚块堆叠

**场景**: 多个锚块堆叠在一起，难以区分个体。

**优先级规则**:
1. 最顶层锚块优先完整标注
2. 底层锚块若可见面积 > 50%，单独标注
3. 若难以区分，整体标注为 `anchor_block_damaged`（提示异常堆叠）

#### 3.3.2 植被遮挡

**场景**: 水草、浮萍等遮挡目标。

| 遮挡比例 | 处理规则 |
|---------|---------|
| < 30% | 正常标注，边界框包含遮挡物 |
| 30% - 70% | 标注可见部分，`difficult=True` |
| > 70% | 标记为 `occlusion`，添加 `difficult` 标记 |

#### 3.3.3 部件间遮挡

**场景**: 锚绳遮挡连接件，或连接件遮挡锚块。

**规则**: 
- 前景部件完整标注
- 背景部件标注可见部分
- 两者均不标记 `difficult`（正常物理遮挡）

### 3.4 藻类覆盖处理

**场景**: 绿色藻类附着在组件表面。

| 覆盖程度 | 颜色变化 | 处理策略 |
|---------|---------|---------|
| 轻微 (< 20%) | 局部发绿 | 正常标注，添加 `algae` 标签 |
| 中度 (20%-50%) | 明显绿色覆盖 | 正常标注，`difficult=True`，添加 `algae` 标签 |
| 重度 (> 50%) | 几乎全绿 | 根据形状推断标注，显著降低置信度权重 |

## 4. 标注质量控制标准

### 4.1 边界框质量标准

#### 4.1.1 紧密包围原则

- **目标**: 边界框应紧密包围目标物体
- **IoU要求**: 与理想框的 IoU ≥ 0.9
- **背景像素占比**: < 5%
- **边缘间隙**: 目标边缘与框边界距离 2-5px

#### 4.1.2 不合格示例

| 问题类型 | 描述 | 示意图 |
|---------|------|-------|
| 过于宽松 | 包含过多背景 | `[目标]   ` 框过大 |
| 过于紧密 | 裁剪目标边缘 | `[目]` 框过小 |
| 偏移 | 中心点偏离目标 | `[ 目标]` 偏心 |
| 旋转不匹配 | 未使用最小外接矩形 | 斜向目标用正框 |

### 4.2 最小像素限制

| 部件类型 | 最小宽度 | 最小高度 | 例外情况 |
|---------|---------|---------|---------|
| 锚绳 | 50px | 50px | 断裂点可放宽至 30px |
| 连接件 | 50px | 50px | 无例外 |
| 锚块 | 50px | 50px | 无例外 |
| 浮筒连接件 | 50px | 50px | 密集区域可放宽至 40px |

**小于限制的处理**:
1. 拒绝标注（推荐）
2. 若目标关键（如断裂点），标注并标记 `difficult=True`
3. 在数据清洗阶段过滤

### 4.3 标注一致性检查

#### 4.3.1 同一目标一致性

- 同一目标在不同图像中应保持相同类别
- 若状态变化（如 normal → loose），需确认时间戳

#### 4.3.2 批量标注审核

每完成 100 张图像，需进行：
1. 随机抽样 10% 复查
2. 检查类别分布是否异常
3. 验证边界框质量

## 5. 负面样本说明

### 5.1 纯背景图定义

**定义**: 图像中完全不包含任何锚固系统组件（锚绳、连接件、锚块、浮筒连接件）。

**常见场景**:
- 纯水面视图
- 仅光伏板特写
- 岸边设施（无锚件）
- 天空/云层

### 5.2 纯背景图标注规则

| 规则项 | 要求 |
|-------|------|
| 标签文件 | 创建空的 `.txt` 文件（0字节） |
| 文件命名 | 与图像同名，如 `bg_001.jpg` → `bg_001.txt` |
| 保留比例 | 占数据集总量的 5-10% |
| 分布要求 | 涵盖不同光照、天气、时间段 |

### 5.3 作用与重要性

1. **抑制误检**: 训练模型识别"无目标"场景，减少误报
2. **提升精度**: 改善 Precision 指标
3. **场景覆盖**: 确保模型见过各种背景

### 5.4 注意事项

- 即使图像边缘有微小目标，也不视为纯背景
- 模糊到无法辨认的目标，按纯背景处理
- 定期更新背景图库，覆盖新场景

## 6. YOLO格式标注规范

### 6.1 文件格式

**文件名**: `{image_name}.txt`

**每行格式**:
```
<class_id> <x_center> <y_center> <width> <height> [difficult]
```

### 6.2 字段说明

| 字段 | 类型 | 范围 | 说明 |
|------|-----|------|------|
| `class_id` | int | 0-15 | 类别ID，见2.1.1 |
| `x_center` | float | 0-1 | 边界框中心点 X 坐标 / 图像宽度 |
| `y_center` | float | 0-1 | 边界框中心点 Y 坐标 / 图像高度 |
| `width` | float | 0-1 | 边界框宽度 / 图像宽度 |
| `height` | float | 0-1 | 边界框高度 / 图像高度 |
| `difficult` | str | optional | 添加 `difficult` 标记难例 |

### 6.3 示例

**图像**: `IMG_20240227_001.jpg` (1920×1080)

**内容**:
```
# 锚绳-松弛 (ID=1), 中心(960, 540), 宽240, 高60
1 0.500000 0.500000 0.125000 0.055556

# 连接件-腐蚀 (ID=7), 中心(480, 270), 宽80, 高80, 难例
7 0.250000 0.250000 0.041667 0.074074 difficult

# 锚块-正常 (ID=8)
8 0.750000 0.750000 0.104167 0.092593
```

### 6.4 质量验证脚本

```python
# validate_labels.py
import os
from pathlib import Path

def validate_yolo_label(label_path, img_width, img_height):
    """验证YOLO格式标签"""
    errors = []
    
    with open(label_path, 'r') as f:
        for line_num, line in enumerate(f, 1):
            parts = line.strip().split()
            
            # 检查字段数
            if len(parts) < 5:
                errors.append(f"Line {line_num}: 字段不足")
                continue
            
            class_id = int(parts[0])
            x, y, w, h = map(float, parts[1:5])
            
            # 检查类别ID
            if not (0 <= class_id <= 15):
                errors.append(f"Line {line_num}: 无效类别ID {class_id}")
            
            # 检查坐标范围
            if not all(0 <= v <= 1 for v in [x, y, w, h]):
                errors.append(f"Line {line_num}: 坐标超出范围")
            
            # 检查尺寸限制
            if w * img_width < 50 or h * img_height < 50:
                errors.append(f"Line {line_num}: 目标尺寸小于50px")
    
    return errors
```

## 7. 标注工具使用指南

### 7.1 推荐工具

| 工具 | 适用场景 | 优点 |
|------|---------|------|
| **LabelImg** | 基础矩形标注 | 轻量，输出YOLO格式 |
| **CVAT** | 团队协作 | 支持项目管理，格式丰富 |
| **Label Studio** | 复杂项目 | 支持多种标注类型 |
| **Roboflow** | 全流程管理 | 自动增强，格式转换 |

### 7.2 LabelImg 快捷键

| 快捷键 | 功能 |
|-------|------|
| `W` | 创建矩形框 |
| `Ctrl+S` | 保存 |
| `D` | 下一张 |
| `A` | 上一张 |
| `Del` | 删除选中框 |
| `Ctrl+Z` | 撤销 |

### 7.3 标注流程

1. **准备阶段**: 加载图像目录，设置自动保存
2. **初标阶段**: 快速浏览，标注明显目标
3. **精修阶段**: 调整边界框，确保紧密包围
4. **审核阶段**: 复查 `difficult` 样本和边界情况
5. **导出阶段**: 导出 YOLO 格式，验证文件完整性

## 8. 质量审核流程

### 8.1 三级审核制度

| 级别 | 执行者 | 检查内容 | 抽样比例 |
|------|-------|---------|---------|
| **自检** | 标注员 | 格式正确性、遗漏检查 | 100% |
| **互检** | 同组标注员 | 类别一致性、边界质量 | 20% |
| **专检** | 质检员 | 疑难样本、整体分布 | 10% |

### 8.2 常见错误清单

- [ ] 类别ID与名称不符
- [ ] 坐标值超出 [0,1] 范围
- [ ] 目标尺寸 < 50px
- [ ] 边界框过于宽松或紧密
- [ ] 遗漏 `difficult` 标记
- [ ] 反射光区域未处理
- [ ] 锚绳线段未转矩形

### 8.3 返修标准

| 错误类型 | 返修要求 |
|---------|---------|
| 严重错误（类别错误） | 100% 返修 |
| 一般错误（框偏移） | 返修率 > 10% 则全量返修 |
| 轻微错误（格式问题） | 自动脚本修正 |
