# 水面光伏场景数据采集与标注优化建议

## 概述

本文档针对"水面光伏区锚固系统检测"项目的特殊挑战——**小目标检测**和**强反光干扰**，提供3条经过验证的数据采集与标注优化建议。这些建议旨在从源头提升数据质量，从根本上改善模型的检测性能。

---

## 建议一：多尺度、多视角、多时相的数据采集策略

### 核心思想

通过系统性的数据采集规划，确保数据集覆盖目标的各种尺度变化和光照条件，特别针对小目标和反光场景进行定向采集。

### 具体实施

#### 1.1 分层采集策略（针对小目标）

```
采集高度分层:
├── 低空层 (5-15m): 用于小目标细节采集
│   └── 锚绳纹理、连接件锈蚀等细微缺陷
├── 中空层 (15-30m): 标准检测高度
│   └── 整体锚固系统状态
└── 高空层 (30-50m): 全景上下文
    └── 锚块位置、阵列整体状态
```

**推荐比例**: 低空:中空:高空 = **4:4:2**

#### 1.2 多视角覆盖（降低遮挡影响）

| 视角类型 | 角度范围 | 主要用途 | 建议比例 |
|---------|---------|---------|---------|
| **垂直俯视** | 80°-90° | 连接件顶部、浮筒状态 | 40% |
| **斜向俯视** | 45°-60° | 锚绳侧面、锚块立体结构 | 40% |
| **水平侧视** | 0°-30° | 水面线位置、淹没情况 | 20% |

#### 1.3 时相采集（应对反光）

```
一天内的采集时间点:
├─ 06:00-08:00 (清晨): 反光较弱，色温偏冷  → 建议 20%
├─ 09:00-11:00 (上午): 反光开始增强        → 建议 20%
├─ 12:00-14:00 (正午): 强反光，挑战最大      → 建议 15% (难例采集)
├─ 15:00-17:00 (下午): 侧向光照             → 建议 25%
└─ 17:00-19:00 (黄昏): 金色光照，低反光      → 建议 20%
```

**关键洞察**: 正午时段虽然反光最强，但正是采集"极端难例"的最佳时机。这些样本对提升模型鲁棒性至关重要。

### 标注配合策略

对于低空采集的高分辨率图像（如4K）：

1. **局部裁剪标注**: 将大图裁剪为 640×640 或 1280×1280 的 patch 进行标注
2. **滑动窗口标注**: 使用滑动窗口（overlap=20%）确保不遗漏边缘目标
3. **尺度归一化**: 在标注工具中统一显示尺度，避免因图像大小不同导致的标注偏差

```python
# 图像裁剪脚本示例
import cv2

def crop_patches(image_path, patch_size=1280, overlap=0.2):
    """滑动窗口裁剪图像"""
    img = cv2.imread(image_path)
    h, w = img.shape[:2]
    stride = int(patch_size * (1 - overlap))
    
    patches = []
    for y in range(0, h - patch_size + 1, stride):
        for x in range(0, w - patch_size + 1, stride):
            patch = img[y:y+patch_size, x:x+patch_size]
            patches.append({
                'image': patch,
                'offset': (x, y),
                'filename': f"{name}_{x}_{y}.jpg"
            })
    return patches
```

## 建议二：偏振滤光与多曝光融合的数据采集技术

### 核心思想

通过硬件手段（偏振镜）和软件手段（多曝光融合）主动抑制水面反光，从源头获取高质量图像，减少后期标注难度。

### 2.1 偏振镜采集方案

#### 为什么有效？

水面反光具有**偏振特性**，使用偏振滤光镜可以：
- 消除 80-95% 的水面镜面反射
- 提升水下目标可见度（淹没的锚绳、锚块）
- 降低动态范围，减少过曝区域

#### 实施方法

**方案A: 无人机搭载偏振相机**
```
推荐设备:
- 大疆 Matrice 300 RTK + 偏振相机模块
- 或: 普通相机 + 可旋转偏振滤镜
```

**方案B: 双相机同步采集**
```
同步采集:
├─ Camera A: 无偏振镜 (保留反光信息)
└─ Camera B: 偏振镜 (消除反光)

后续处理:
- 用 Camera B 进行标注（清晰可见）
- 用 Camera A 训练反光鲁棒性
- 或: 图像融合获取 HDR 效果
```

**方案C: 单相机多角度偏振**
```
同一位置拍摄3张:
├─ 偏振角 0°  (最大反光)
├─ 偏振角 45° (中等反光)
└─ 偏振角 90° (最小反光)

标注策略:
- 优先标注 90° 图像
- 将标注映射到 0° 和 45° 图像
- 三图同时加入训练集，增强泛化能力
```

### 2.2 多曝光融合采集

针对极端光照对比场景（亮处过曝+暗处欠曝）：

```
采集序列 (包围曝光):
├─ EV -2: 欠曝图像 (保留亮部细节)
├─ EV  0: 正常曝光
└─ EV +2: 过曝图像 (保留暗部细节)

融合算法:
- Mertens 融合 (OpenCV 内置)
- 或: 手动选择最佳曝光区域
```

**Python实现**:

```python
import cv2

def capture_exposure_bracket(camera, ev_steps=[-2, 0, 2]):
    """采集曝光包围序列"""
    images = []
    for ev in ev_steps:
        camera.set(cv2.CAP_PROP_EXPOSURE, ev)
        ret, frame = camera.read()
        if ret:
            images.append(frame)
    return images

def fuse_exposures(images):
    """融合多曝光图像"""
    merge_mertens = cv2.createMergeMertens()
    fused = merge_mertens.process(images)
```

### 2.3 标注配合策略

**偏振图像标注流程**:

1. **主标注**: 在偏振图像（反光抑制）上进行精确标注
2. **映射验证**: 将标注框映射到非偏振图像，验证位置准确性
3. **难例标记**: 在非偏振图像中因反光导致模糊的目标，标记 `difficult=True`
4. **对比学习**: 同一目标的偏振/非偏振版本同时加入训练集

**质量提升预期**:
- 反光场景下的标注准确率: ↑ 30-40%
- 目标检出率 (Recall): ↑ 15-25%
- 标注效率: ↑ 50%（因图像更清晰）

## 建议三：主动难例挖掘与定向增强的标注策略

### 核心思想

不被动等待难例出现，而是主动设计和采集难例场景，并通过精细化的标注策略最大化难例的训练价值。

### 3.1 定向难例采集清单

基于对模型失效模式的预判，定向采集以下难例：

| 难例类型 | 采集方法 | 目标数量 | 标注重点 |
|---------|---------|---------|---------|
| **微小断裂点** | 近距离拍摄锚绳断裂细节 | 每张图1-2个 | 精确标注断点位置，标记 `difficult` |
| **部分淹没锚块** | 不同水位时重复拍摄 | 每个锚块3-5张 | 标注可见部分，估算淹没比例 |
| **反光遮挡连接件** | 正午时段多角度拍摄 | 占数据集 10% | 标注可见轮廓，标记 `glare` |
| **密集锚绳区域** | 阵列中心区域俯拍 | 占数据集 5% | 逐根标注，允许轻微重叠 |
| **藻类重度覆盖** | 夏季藻类爆发期采集 | 占数据集 8% | 标注实际边界，标记 `algae` |
| **运动模糊** | 有风天气/快速飞行采集 | 占数据集 5% | 根据上下文推断标注 |

### 3.2 精细化标注策略

#### 分层标注体系

对于复杂目标，采用分层标注策略：

```
标注层级:
├─ Level 1: 粗框 (Coarse Box)
│   └── 快速包围目标，用于预训练
│
├─ Level 2: 精框 (Tight Box) [标准]
│   └── 紧密包围，IoU > 0.9，用于主要训练
│
└─ Level 3: 部件级 (Part Level)
    └── 标注子部件（如连接件的螺母、垫片）
        用于细粒度分析和故障定位
```

#### 属性标签扩展

在标准YOLO格式基础上，扩展属性标签：

```
扩展格式:
<class_id> <x> <y> <w> <h> <attributes>

属性字段（可选）:
- difficult: 难例标记
- confidence: 标注置信度 (0-1)
- occlusion: 遮挡比例 (0-1)
- visibility: 可见度 (clear/fuzzy/hidden)
- glare: 反光影响 (none/medium/severe)

示例:
1 0.500 0.500 0.100 0.050 difficult occlusion=0.3 visibility=fuzzy
```

### 3.4 难例挖掘闭环

建立"训练→验证→难例发现→补充标注"的闭环：

```
流程:
1. 初始训练
   └─ 使用现有数据集训练模型
   
2. 错误分析
   └─ 在验证集上运行推理
   └─ 收集 False Negative 和 False Positive
   
3. 难例分类
   ├─ 反光导致的漏检
   ├─ 小目标漏检
   ├─ 相似类别误检
   └─ 遮挡导致的漏检
   
4. 定向补充
   └─ 针对难例类型采集新数据
   └─ 或：从现有数据中挖掘相似样本
   
5. 迭代训练
   └─ 加入难例重新训练
   └─ 返回步骤2
```
